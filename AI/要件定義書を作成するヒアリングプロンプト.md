# AIを使用して要件定義書を作成するヒアリングプロンプト

MVPを意識した要件定義を行なってくれるよう作成したプロンプトになっています。

## 基本的なヒアリングプロンプト

```
私はアプリを作りたいと思っていますが、要件定義の方法がよく分かりません。
あなたに要件定義のプロとして、私にヒアリングをしていただけますか？
以下の流れで進めてください。

1. まず、私がどんなアプリを作りたいか、簡単に聞いてください
2. そのアプリについて、目的、ユーザー、利用シーンなどを段階的に質問してください
3. 必要な機能について、「最初に絶対必要な機能」と「後で追加したい機能」を分けて整理してください
4. 最後に、ヒアリング内容を整理して要件定義書として提示してください

重要な点：
・まず最小限の機能（MVP）で動くアプリを作りたいです
・質問は一度に1つずつ、分かりやすい言葉で聞いてください
・私はプログラミング初心者なので、技術的な質問は避けて、使う人の立場で考えられる質問をお願いします

それでは、最初の質問をしてください。
```

## データ設計に関する追加プロンプト

```
このアプリで扱うデータについて、以下の点に配慮した実装計画を立ててください：

1. データの保存方法（どんな情報をどう整理して保存するか）
2. データ同士の関連性（例：ユーザーと投稿の紐付けなど）
3. データを削除する時の注意点（例：ユーザーを削除したら投稿も消すかなど）
4. 大量のデータを扱っても遅くならない工夫

プログラミング未経験者でも理解できるように、専門用語を使わずに説明してください。
```

## CLAUDE.md作成用プロンプト

```
以下の要件定義内容をもとに、Claude Code開発用のCLAUDE.mdファイルを作成してください。

■ 要件定義内容
[第3章で整理した内容を記載]

CLAUDE.mdには以下の要素を含めてください：

1. プロジェクト概要
2. ターゲットユーザー
3. アプリのゴール（WHO/WHAT/HOW）
4. 主要機能（MVP/将来機能の分類）
5. 技術的制約・要望
6. UI/UX方針
7. 開発の進め方

マークダウン形式で、Claude Codeが理解しやすい構造にしてください。
また、今後の開発で参照しやすいよう、具体的で明確な表現を心がけてください。
```

## CLAUDE.md更新用プロンプト

### 新機能追加時の更新

```
新しい機能を追加する際は、CLAUDE.mdも更新します。

以下の新機能を追加しました：
[追加した機能の説明]

CLAUDE.mdファイルを更新して、この変更を反映してください。
特に「主要機能」セクションと「開発進捗」セクションの更新をお願いします。
```

### 要件変更時の更新

```
開発を進める中で、以下の要件変更がありました：

- 変更前：[元の要件]
- 変更後：[新しい要件]
- 変更理由：[なぜ変更したか]

CLAUDE.mdファイルを更新して、この変更を反映してください。
```
