## ページ 1

Mac ローカル環境構築マニュアル
はじめに：ローカル開発環境の構築
このマニュアルでは、ご自身のMacPC上に、Webアプリケーションを開発するための環境を構築
する手順を解説します。
これまでの開発環境：GitHub Codespaces
これまでは、GitHub Codespacesというサービスを利用して、Webアプリケーションの開発を行
ってきました。
GitHub Codespacesは、Webブラウザさえあれば、どこでもすぐに開発を始めることができる非
常に便利なクラウド開発環境です。必要なツールや設定が予め用意された仮想的なコンピュータ
を、GitHubがインターネットの向こう側で貸し出してくれる、とイメージすると良いでしょう。
ターミナル操作やファイルの編集、アプリケーションの実行まで、すべてブラウザのタブの中で完
結するため、開発を始める上でのハードルを大きく下げてくれました。
ローカル開発環境とCodespacesの違い
これから私たちが構築しようとしているのは、「ローカル開発環境」です。
これは、Codespacesのようなインターネット上の仮想的なコンピュータではなく、今まさにあな
たが操作しているご自身のMacPCそのものの上に、開発に必要なツール群を一つずつインストー
ルして作り上げる環境を指します。

## ページ 2

1. オフラインでの作業
一度環境を構築してしまえば、インターネットに接続していない状態でも、コードの編集や
アプリケーションの動作確認（一部機能を除く）が可能です。
2. 実行速度と安定性
PCのスペックにもよりますが、一般的にローカル環境でのファイル操作やアプリケーション
のビルドは、ネットワーク遅延の影響を受けないため、Codespacesよりも高速で安定して
いる場合があります。
3. 環境の永続性と自由なカスタマイズ
Codespacesは一定時間で停止しますが、ローカル環境はあなたがPCをシャットダウンする
まで動き続けます。また、将来的にあなたが使いたいと思ったツールを、制約なく自由にイ
ンストールし、自分だけの最適な開発環境を育てていくことができます。
このマニュアルを通じて、あなたはこのローカル開発環境の構築プロセスを学び、開発者としてよ
り自立したスキルを身につけることになります。

3. Node.jsのインストールと設定
3-1. 専門用語解説：Node.js およびNode.jsの管理ツールの
役割
Node.js とは？
GitHub Codespacesで
npm run dev
というコマンドを実行すると、開発サーバーが起動してア
プリケーションが動きましたね。あの時、あなたの代わりに

簡単に言うと、JavaScriptという言語を元にしたプログラムをPC上で動かせる状態を整えてくれ

## ページ 12

通常、JavaScriptはWebブラウザの中で動く言語ですが、Node.jsをPCにインストールすること
で、ターミナルのような場所でもJavaScriptのプログラム（Next.jsアプリもその一つです）を動
かせるようになります。
CodespacesにはこのNode.jsが最初からインストールされていたため、私たちはその存在を意識
する必要がありませんでした。しかし、ローカル環境では、この心臓部を自分自身でPCに導入して
あげる必要があります。
2. nvm (Node Version Manager) とは？
では、Node.jsを直接PCにインストールすれば良いのでしょうか？それでも動きますが、私たちは
より賢い方法としてnvm というツールを経由します。
nvmは、「Node.jsのバージョンを管理するための専門ツール」です。
Web開発の世界は変化が速く、Node.jsにも様々なバージョンが存在します。将来、あなたが別の
プロジェクトに参加した際、「このAプロジェクトはNode.jsのバージョン18で動かして」「こちら
のBプロジェクトは最新のバージョン20が必要」といった状況が必ず発生します。
もしNode.jsを直接PCにインストールしてしまうと、このバージョン切り替えが非常に面倒になり
ます。
nvmは、まるでテレビのリモコンでチャンネルを切り替えるように、「今はバージョン18を使いま
す」「次はバージョン20に切り替えます」といった操作を、コマンド一つで簡単に行えるようにし
てくれます。
最初にこの一手間をかけておくことで、将来の面倒な作業を回避できるのです。
