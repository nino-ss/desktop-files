3-1. 専門用語解説：Node.js と nvm の役割
1. Node.js とは？
GitHub Codespacesで
npm run dev
というコマンドを実行すると、開発サーバーが起動してア
プリケーションが動きましたね。あの時、あなたの代わりに
npm
というコマンドを解釈し、
Next.js（JavaScript）のプログラムを実行してくれていたもの、それがNode.js です。
簡単に言うと、JavaScriptという言語を元にしたプログラムをPC上で動かせる状態を整えてくれ
るソフトです。Next.js を動かすためにもNode.jsが必要です。
通常、JavaScriptはWebブラウザの中で動く言語ですが、Node.jsをPCにインストールすること
で、ターミナルのような場所でもJavaScriptのプログラム（Next.jsアプリもその一つです）を動
かせるようになります。
CodespacesにはこのNode.jsが最初からインストールされていたため、私たちはその存在を意識
する必要がありませんでした。しかし、ローカル環境では、この心臓部を自分自身でPCに導入して
あげる必要があります。
2. nvm (Node Version Manager) とは？
では、Node.jsを直接PCにインストールすれば良いのでしょうか？それでも動きますが、私たちは
より賢い方法としてnvm というツールを経由します。
nvmは、「Node.jsのバージョンを管理するための専門ツール」です。
Web開発の世界は変化が速く、Node.jsにも様々なバージョンが存在します。将来、あなたが別の
プロジェクトに参加した際、「このAプロジェクトはNode.jsのバージョン18で動かして」「こちら
のBプロジェクトは最新のバージョン20が必要」といった状況が必ず発生します。
もしNode.jsを直接PCにインストールしてしまうと、このバージョン切り替えが非常に面倒になり
ます。
nvmは、まるでテレビのリモコンでチャンネルを切り替えるように、「今はバージョン18を使いま
す」「次はバージョン20に切り替えます」といった操作を、コマンド一つで簡単に行えるようにし
てくれます。
最初にこの一手間をかけておくことで、将来の面倒な作業を回避できるのです。

3-3. 重要：環境変数「Path」の設定について
さて、ここがこのマニュアルで最も重要なポイントです。
nvmをインストール後は、そのままターミナルから"nvm"というコマンドを使ってコントロールし
ます。
しかし、nvmをインストールしただけでは、まだPCはnvmを利用できません。
VSCodeで開いたGit Bashで
nvm
と打っても、「そんなコマンドは知らない(command not
found)」と返されてしまうでしょう。
これは、PCに「Path（パス）が通っていない」という状態です。
「Pathを通す」とは何か？
これを理解するために、PCを「巨大な図書館」、ターミナルを「司書さん」だと想像してみてく
ださい。
1. あなたが司書さん（ターミナル）に「nvm という本を持ってきて」と頼みます。（nvm コ
マンドを実行）
2. 司書さんは、図書館のどこにどんな本があるかをすべて把握しているわけではありません。
司書さんは、いつも本を探しに行くと決めている、いくつかの特定の棚だけを見に行きま
す。
3. nvm という本は新しく入荷したばかりで、新しい棚に置かれています。
4. 司書さんは、決まった棚にnvm という本がなかったので、「そんな本はありません」とあな
たに報告します。（command not found ）
ここで必要になるのが、「Pathを通す」という作業です。
これは、司書さんに「これからはあの新しい棚も、本を探しに行く対象の棚リストに追加しておい
てね」と、探すべき場所のリストを更新してあげる作業なのです。
技術的に言うと、Windowsはコマンドが実行された時、環境変数「Path」に登録されているフォ
ルダの中を順に探しに行き、一致する名前の実行ファイルがあればそれを実行します。「Pathを通
